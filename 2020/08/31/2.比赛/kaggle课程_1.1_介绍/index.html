竞赛结构

一般来说，竞赛的内容各不相同，有的是要求你处理文本，有的是要求你处理图像，有的是要求你预测未来广告的收益。但是其实他们的结构都是相同的，如下所示：

  • Data
    • 数据是主办方给我们的培训材料，我们需要使用data来产生我们的解决方案。
    • 数据的展示形式各不相同，有的是csv数据，有的是txt文件，有的是图片或者数据库格式
    • 数据通常有描述性文字，告诉你哪些特征可以被提取
    • 有时可以使用额外的数据集来提高模型性能表现，比如CV分类问题如果使用现有独立数据集性能表现会大大提升
  • Model
    • 我们在竞赛中建立的就是模型
    • 最好不要把模型看成是特定的算法，而是一些把数据转换为答案的东西
    • 模型要有两个特征：是可重复的以及给出最好的预测结果的
    • 事实上，模型包含了大量的算法、手工制作的特征、利用了各种library，是庞大包含了各种组件的，如下
  • Submission
    • 为了和其他人比较模型的优劣，我们需要将预测结果发送到服务器。通常你只需要提交结果而不需要来源和模型。
  • Evaluation
    • 评价函数是一种输入是（你的答案+正确答案）输出是（评价分数）的函数。通常情况下，有很多这样的函数可以用，比赛具体使用哪一种会在说明中指出。你需要注意并迎合它。
  • Leaderboard
    • 但是很多时候我们不需要对比分本身感兴趣，而是需要对你与其他对手之间的表现对比感兴趣，也就是排名榜感兴趣。
    • 但是因为有些人可以通过大量提交来获得对自己有利的结果，比赛的leaderboard分类公开leaderboard和私人leaderboard，私人board中的数据在最终评测前不公开。
  • Routine
    • 标准的竞赛流程是：分析数据 - 改进模型 - 提交数据 - 看排行榜分数 - 重复上边的动作若干次 - 最终提交。在加上了私人数据集最终评估的时候，允许你选择两个最终结果，要明智选择，公开排名有时候是被误导的。

竞赛的意义

  • 主要原因是竞赛是学习的大好机会，你会尝试与其他队员沟通，尝试新的方法，得到了很多经验
  • 竞赛通常给你提高了非平凡的问题和最优质的解决方案
  • 很好的出名的机会。你会得到某种科学界内部的人员框架结构，并获得不错的工作机会。
  • 赢钱

常见算法

  • 线性算法
    • 逻辑回归和svm都是线性算法
    • 线性算法的中心思想是用一个平面将数据一分为二
    • 线性算法对稀疏高维数据特别好
    • 线性算法的局限性：两个环形分布的点集不能被如此简单的一个方法分离，如下这种情况使用线性算法就会失败
  • 树结构算法
    • 决策树、随机森林、GBDT、XGBoost
    • 决策树图示
    • 通常树结构的算法非常强大，是处理表格数据的良好的默认方法。
    • 树结构上算法的不足；因为它有大量分别的步骤，所以它很难捕捉线性相关性，如下，灰色线是树结构分类器，它可能不是最准确的近决策边界（红色是潜在误分类点），绿色线是线性分类边界
  • KNN
    • KNN是k-最邻近的缩写，knn的直觉表示很简单就是相近的物体更可能是同类。
  • 神经网络算法
    • 神经网络可以理解为一个可以产出光滑决策曲线的分类器（对比于决策树）
    • 神经网络尤其擅长对图像、声音、文本、序列的处理。

算法选择上的“没有免费午餐定理”

没有哪个算法在对待所有任务时，其性能表现会好于其他所有算法。

因为每个算法都有其依赖的数据和假设，任务切换了假设也就切换了算法就不会是表现最好的那个。

这也就意味着我们不能以一套算法走遍天下，不同的任务必然得构建不同的算法，

如下所示，同样的数据集若使用不同的算法会得到不同的决策边界和性能表现

常见算法结论

  • 没有哪个会优于其他所有算法
  • 线性算法的本质就是将数据分割成两个超平面
  • 树结构的算法将决策面分成不同的box，然后在每个box里边使用常量分割
  • knn算法基于“近处物体可能有相同标签”的假设
  • 神经网络较难解释，反正会产生光滑的决策边界
  • 最有力的方法是梯度提振决策树和神经网络算法，但是我们不该低估knn和线性算法因为有时他们的表现更好。
