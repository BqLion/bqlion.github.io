2.入门案例IOC

2.2目标类

  • 提供UserService接口和实现类

    • UserService.java – 接口

  • 获得UserService实现类的实例

    • UserServiceImpl.java –实现

  • 配置文件

    • 位置:任意,一般在classpath下
    • 名称:任意,开发中常用applicationContext.xml
    • 内容:添加schema约束
    • beans.xml
      • <bean id=”userServiceId” class=”a_ioc.UserServiceImpl”></bean>
      • id :bean的名称
      • class:bean的全类型地址

  • 测试

    • @Test 是测试入口,类似于main()函数的程序接口

    • Spring之前和Spring之后对比

      • 之前

        • @Test
    public void demo01(){
        //之前开发
          UserService userService = new UserServiceImpl();
        userService.addUser();
}

      • 之后

        •  @Test
    public void demo02(){
        //从spring获得
        String xmlPath = "a_ioc/beans.xml";
        ApplicationContext applicationContext = new ClassPathXmlApplicationContext(xmlPath);

        //从spring获得内容,不用自己new
        UserService userService = (UserService)applicationContext.getBean("userServiceId");
        userService.addUser();
    }

    • 通过application和注解和一些列流程,实现了一个”new”

    • 回避了new,就回避了代码之间的紧耦合

3.DI

DI: dependency injection依赖注入

依赖:一个对象需要使用另一个对象

  • is a :自己是(继承)

  • has a:有一个(成员变量,依赖)

    • 举例

      class B{

      ​ private A a}

      B是依赖于A的

注入:通过setter方法对另一个对象实例设置

  • 例如：class BookServiceImpl{

    ​ BookService bookService = new BookServiceImpl() //创建serivce实例

    ​ BookDao bookDao = new BookDaoImpl() //创建DAO实例

    ​ bookService setBookDao(bookDao)} //将DAO设置给service，DI

3.1目标类

  • 创建BookService接口和实现类

  • 创建BookDao接口和实现类

    –接口提供了一个层次的抽象，修改底层实现的时候上层不用改太多

  • 将DAO和Service配置xml文件

  • 使用api测试

程序中依赖注入的过程

public class BookServiceImpl implements BookService{
    private BookDao bookDao;
    public void setBookDao(BookDao bookDao){
        this.bookDao = bookDao;
    }

    @Override
    public void addBook(){
        this.bookDao.addBook();
    }
}

–通过beans.xml文件，将BookDao注入到BookService里边

  <bean id="bookServiceId" class="b_di.BookServiceImpl">
        <property name = "bookDao" ref ="bookDaoId"></property>
    </bean>

    <bean id = "bookDaoId" class ="b_di.BookDaoImpl"></bean>

Junit测试

public class TestIoc {
    @Test
    public void demo01(){
        String xmlPath = "b_di/beans.xml";
        ApplicationContext applicationContext = new ClassPathXmlApplicationContext(xmlPath);
        BookService bookService  = (BookService) applicationContext.getBean("bookServiceId");

        bookService.addBook();
    }
}
